# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Automated release creation with GitHub Releases
- Changelog generation following Keep a Changelog format
- Clean Commit (Conventional Commits) parsing
- Semantic versioning with configurable bump rules
- Customizable commit type to changelog section mapping
- Support for version prefixes and prerelease suffixes
- Commit and scope exclusion filters
- Rich action outputs including version info and categorized commit counts
- Dry-run mode for testing
- Tag-only mode for creating tags without releases
- Draft release support
- Prerelease support
- Customizable release name templates
- Comprehensive documentation with examples
- Example workflow for self-releasing
